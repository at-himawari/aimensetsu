## 2. 優先度と進め方

### 2.1 開発優先順位
1. 仕様の最終確定
2. バックエンド基盤実装
3. 外部連携実装
4. フロントエンド実装
5. テスト実装
6. インフラ調整と運用準備

### 2.2 進め方の基本方針
- 先に DB と API の土台を固める
- 外部連携はモック可能な形で切り離して実装する
- 課金、認証、クレジットは早めに自動テスト対象へ入れる
- Cloud Run の起動遅延を前提に UX とタイムアウトを調整する
- AWS の新規リソース追加は手作業ではなく AWS CDK で管理する
- データベースは既存環境を利用し、新規作成タスクは持たない
